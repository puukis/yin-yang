# Yin-Yang 0.1.0-alpha.1

Initial public alpha release for Yin-Yang. This cut turns the repo into a tagged, installable project with automated host and client artifacts, a real CI matrix, and a release pipeline that survives generic GitHub runners without requiring NVIDIA or Wayland graphics libraries at link time.

## New Features

- Ship the first public alpha release with packaged Linux and Windows host binaries plus packaged macOS client binaries. (57f89ac)
- Add clap-based entrypoints and parser tests for the host and client binaries. (c317203)
- Add repository health files, CI, Dependabot, release automation, and tagged packaging metadata. (62388b4)
- Support QUIC datagram video transport, remote cursor transport, HDR D3D11 capture on Windows, and the macOS VideoToolbox plus Metal client path. (3391e19)

## Bug Fixes

- Fix Linux and macOS CI failures uncovered while hardening the release lane. (cdb64c7)
- Fix the remaining GitHub clippy failures across the host and client builds. (2b3dbdc)
- Make client input capture dependencies macOS-only so Linux server packaging no longer drags in X11 and XI client libraries. (6140535)
- Runtime-load NVENC, CUDA, and GBM libraries on Linux so release builds and binaries do not require those GPU libraries at link time. (6cd1368)
- Fix macOS input capture, renderer startup ordering, frame presentation stability, and locked-cursor mouse delta handling during the pre-release hardening pass. (31b6c9e)

## Changed

- Refresh the README, contribution guide, metadata, and repo presentation so the project reads like a usable alpha instead of a local experiment. (c4ef857)
- Prepare the alpha release cut across crate versions and the top-level changelog. (809ca2d)
