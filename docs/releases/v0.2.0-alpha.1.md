# streamd 0.2.0-alpha.1

streamd 0.2.0-alpha.1 hardens the QUIC video path against isolated datagram loss and adds the telemetry needed to adapt the stream without a reconnect. This cut also switches GitHub releases to curated repo-owned markdown so release notes are reviewed and versioned alongside the code.

## New Features

- Protect QUIC video fragments with XOR parity FEC so the macOS client can recover one lost fragment per protection group without waiting for a keyframe.
- Add client and server telemetry for send pressure, fragment recovery, decode/render queueing, and encoder targets, then feed that data into adaptive bitrate and framerate control.
- Add reconnect-aware client session management plus client CLI controls for adaptive streaming bounds, including requested max/min FPS and bitrate floors and ceilings.

## Bug Fixes

- Rate-limit client `RequestIdr` recovery so unrecoverable loss requests a fresh keyframe without flooding the control stream.
- Respect Quinn's live QUIC datagram size when fragmenting frames so PMTU changes do not manufacture oversized datagrams.

## Changed

- Extend the video datagram header and protocol version so parity and telemetry metadata travel across client and server together.
- Publish GitHub release titles and bodies from checked-in `docs/releases/<tag>.md` files instead of generated boilerplate.

## Infra

- Add the `scripts/generate_release_notes.py` scaffold generator, tests for it, and the release workflow plumbing needed to publish curated release notes.
